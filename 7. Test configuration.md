# 7. TEST CONFIGURATION

Mục này mô tả **cấu hình cụ thể** áp dụng khi thực thi kiểm thử (khác với mục 6 chỉ liệt kê môi trường chung). Dùng để **tái lập** cùng điều kiện giữa các thành viên và giữa các vòng test.

---

## 7.1 Hardware configuration

### 7.1.1 HW_CFG_001 — Máy trạm kiểm thử chính (laptop/PC)

**Mô tả ngắn:** Máy dùng để chạy `DATN_api-main`, `DATN_web_client-main`, trình duyệt và công cụ gọi API trong suốt đợt test.

| Setting | Value | Remark |
|---------|-------|--------|
| RAM khả dụng cho dev/test | ≥ 8 GB (khuyến nghị 16 GB) | Đóng bớt ứng dụng nặng khi chạy DB + API + web |
| Độ phân giải màn hình (web UI) | 1920 × 1080 (tham chiếu) | Ghi thêm 1366×768 nếu test thêm độ phân giải thấp |
| Kết nối mạng | Wi-Fi / LAN ổn định | Bắt buộc khi test VNPay, GHN, S3 |
| Nguồn điện | Cắm adapter khi test dài | Tránh mất session giữa chừng |

### 7.1.2 HW_CFG_002 — Thiết bị bổ sung (tuỳ chọn)

**Mô tả ngắn:** Dùng khi nhóm kiểm thử responsive hoặc luồng thanh toán QR trên mobile.

| Setting | Value | Remark |
|---------|-------|--------|
| Điện thoại / tablet | 0–1 thiết bị | Cùng mạng LAN với máy dev hoặc truy cập URL public staging |
| Trình duyệt trên mobile | Chrome / Safari bản mới | Ghi rõ model nếu nộp báo cáo |

*Nếu chỉ test trên desktop:* ghi **N/A** cho HW_CFG_002.

---

## 7.2 Software configuration

### 7.2.1 SW_CFG_001 — Backend API (`DATN_api-main`)

**Mô tả ngắn:** Biến môi trường và chế độ chạy thống nhất khi thực thi TC API và tích hợp.

| Setting | Value | Remark |
|---------|-------|--------|
| `NODE_ENV` | `development` hoặc `test` | Không dùng production secret trên máy cá nhân |
| `PORT` (API) | `3000` (hoặc giá trị nhóm thống nhất) | Khớp URL trong Postman và CORS web |
| `DATABASE_URL` | Chuỗi PostgreSQL trỏ **DB test** | Tách khỏi DB production nếu có |
| Múi giờ hệ thống | `Asia/Ho_Chi_Minh` (hoặc UTC nếu nhóm thống nhất) | Quan trọng cho khuyến mãi theo ngày, token hết hạn |
| Log level | Theo `nestjs-pino` (ví dụ `debug` khi dev) | Giảm `info` khi cần log gọn cho báo cáo |

### 7.2.2 SW_CFG_002 — Web client (`DATN_web_client-main`)

**Mô tả ngắn:** Cách gọi API và ngôn ngữ giao diện khi test thủ công.

| Mode / Setting | Value | Remark |
|----------------|-------|--------|
| URL API (proxy / env) | `http://localhost:<PORT_API>` hoặc biến `NEXT_PUBLIC_*` nhóm thống nhất | Khớp backend đang chạy |
| Cổng dev Next.js | `8000` (theo `package.json`) | Ghi lại nếu đổi |
| Ngôn ngữ UI (i18n) | Ví dụ `vi` hoặc `en` | Chọn **một** ngôn ngữ làm chuẩn cho bộ TC hoặc ghi TC song ngữ |
| Trình duyệt chuẩn | Chrome (bản mới nhất trong phạm vi đồ án) | Có thể thêm một trình duyệt thứ hai để smoke |

### 7.2.3 Automation test tool configuration

**Mô tả ngắn:** Cấu hình khi nhóm dùng Jest (API) hoặc Playwright/Cypress (web).

| Setting | Value | Remark |
|---------|-------|--------|
| Jest config | `test/jest.json` (trong `DATN_api-main`) | Lệnh: `npm test` |
| Môi trường khi chạy Jest | `NODE_ENV=test` (nếu có trong script) | Mock DB/HTTP theo từng `*.spec.ts` |
| Playwright / Cypress (tuỳ chọn) | Đường dẫn file cấu hình do nhóm thêm | Ghi base URL trỏ `http://localhost:8000` |
| Headless / headed | `headless: true` trên CI; `false` khi debug local | Ghi trong README nhóm |

*Nếu chưa có automation:* ghi **Chưa áp dụng** và chỉ giữ Jest mặc định của Nest khi có chạy.

## Ghi chú

- Mỗi vòng test nên **ghi snapshot**: commit hash + bảng cấu hình SW_CFG đã dùng (copy vào báo cáo hoặc issue).
- Khi đổi `DATABASE_URL` hoặc URL API, **chạy lại** smoke TC trước khi tiếp tục bộ test dài.
