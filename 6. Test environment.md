# 6. TEST ENVIRONMENT

*(Trong biểu mẫu Excel gốc có thể ghi **Test enviroment** — đây là cùng một mục.)*

Mục này liệt kê **phần cứng, phần mềm và hạ tầng** dùng để kiểm thử hệ thống DATN (`DATN_api-main`, `DATN_web_client-main`, cơ sở dữ liệu). Số lượng máy và thời gian sử dụng: **theo thời gian đồ án** (từ khi thiết lập môi trường đến khi kết thúc vòng test/nộp bài).

---

## 6.1 Hardware

| Thiết bị / tài nguyên | Số lượng gợi ý (nhóm 4 người) | Thời gian sử dụng | Ghi chú |
|----------------------|----------------------|-------------------|---------|
| Máy tính cá nhân (PC/laptop) | 4 | Suốt giai đoạn phát triển và test | Mỗi thành viên có thể chạy API + web local; cấu hình tối thiểu: **8 GB RAM** (khuyến nghị **16 GB** khi chạy DB + IDE + browser cùng lúc) |
| Màn hình | Theo nhu cầu | — | — |
| Kết nối mạng ổn định | 1 kênh / người | Khi test tích hợp VNPay, GHN, Recurly, S3 | Cần cho sandbox và upload media |
| Điện thoại / tablet (tuỳ chọn) | 0–2 | Kiểm tra responsive web TMĐT | Không bắt buộc nếu chỉ scope desktop |

---

## 6.2 Software

| Phần mềm | Phiên bản / ghi chú | Mục đích |
|----------|---------------------|----------|
| Hệ điều hành | Windows / macOS / Linux (64-bit) | Dev và chạy test |
| Node.js | Khớp với `package.json` (ví dụ **22.x LTS**) | Chạy API NestJS và build Next.js |
| Trình quản lý gói | npm (mặc định); có thể yarn/pnpm nếu nhóm thống nhất | Cài dependency |
| PostgreSQL | Khớp phiên bản hỗ trợ của Prisma trong repo | CSDL cho `DATN_api-main` (provider `postgresql` trong `schema.prisma`) |
| Trình duyệt | Chrome / Edge / Firefox (bản mới nhất trong phạm vi đồ án) | Kiểm thử UI web |
| Git | Bản ổn định | Clone, nhánh, build theo commit kiểm thử |
| IDE / editor | VS Code, WebStorm, v.v. | Sửa mã và chạy test |
| Docker (tuỳ chọn) | Theo `docker-compose` nếu nhóm dùng | Đồng nhất DB hoặc Redis nếu có trong stack |

**Chạy ứng dụng (tham khảo):**

| Thành phần | Lệnh / cổng (mặc định thường gặp) |
|------------|-------------------------------------|
| API | `npm run dev` (NestJS) — cổng **3000** (hoặc `PORT` trong `.env`) |
| Web | `npm run dev` (Next.js) — cổng **8000** theo `package.json` client |
| Swagger | Trình duyệt tại `http://localhost:<PORT>/api` (khi bật backend) |

Biến môi trường (`.env`, `.env.example`): **DATABASE_URL**, khóa **JWT**, **CORS**, khóa **AWS S3**, **VNPay**, **GHN**, **Recurly**, **Redis** (nếu bật) — chỉ dùng **giá trị test/sandbox**, không commit secret lên kho công khai.

## 6.3 Infrastructure

Công cụ và dịch vụ hỗ trợ quản lý test, lỗi và nỗ lực (điều chỉnh theo thực tế nhóm).

| Purpose | Tool | Vendor / In-house | Version (ghi khi dùng) |
|---------|------|-------------------|-------------------------|
| Quản lý lỗi (defect log) | GitHub Issues / GitLab Issues / Google Sheet | Theo kho nhóm | — |
| Theo dõi tiến độ test | GitHub Projects / Trello / Notion / Excel | In-house | — |
| Gọi API thủ công | Postman / Thunder Client / Insomnia | Bên thứ ba (miễn phí) | Bản cài đặt hiện tại |
| Tài liệu API | Swagger UI (tích hợp NestJS) | Sinh từ mã | Theo bản build |
| Kiểm thử đơn vị (API) | Jest | Open source | Theo `package.json` |
| Kiểm thử tự động web (tuỳ chọn) | Playwright / Cypress | Open source | Theo bản cài nhóm |
| CI (tuỳ chọn) | GitHub Actions | GitHub | Theo workflow |

**Tài khoản / dịch vụ ngoài (chỉ môi trường test):**

- Sandbox **VNPay**, **GHN**, **Recurly** (nếu dùng), bucket **S3** hoặc tương đương — đảm bảo key có **hạn mức** và không dùng chung production.

---

## Ghi chú

- Môi trường test **không nhất thiết** trùng production; cần **ghi rõ** URL API và web (localhost hoặc staging) trong báo cáo kiểm thử.
- Khi thiếu sandbox (ví dụ khóa hết hạn), ghi **Blocked** trên TC và không kết luận sai lỗi logic nội bộ cho đến khi có môi trường hợp lệ.
