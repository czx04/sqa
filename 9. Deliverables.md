# 9. DELIVERABLES

Mục này liệt kê **sản phẩm bàn giao** của hoạt động kiểm thử đồ án **DATN**. Ngày giao là **mục tiêu**; cập nhật thực tế khi nộp bài. Tham chiếu thêm kế hoạch dự án (Project Plan) nếu nhóm có file riêng.

**Product ID** dùng để truy vết trong hoạt động review / nghiệm thu (theo tinh thần biểu mẫu FSOFT).

---

## Bảng sản phẩm bàn giao

| No | Product ID | Deliverables | Ngôn ngữ | Ngày giao (dự kiến / thực tế) |
|----|--------------|--------------|----------|-------------------------------|
| 1 | DATN-TP-001 | Test Plan (bộ tài liệu theo mẫu FSOFT / file `.md` trong thư mục SQA) | Tiếng Việt | dd/mm/yyyy |
| 2 | DATN-UT-001 | Bộ Unit Test Case / script (tham chiếu `*.spec.ts` và mô tả ngắn nếu có) | Tiếng Việt / comment code English theo repo | dd/mm/yyyy |
| 3 | DATN-IT-001 | Integration Test Cases / script (API + DB + dịch vụ ngoài; Postman collection hoặc bảng TC) | Tiếng Việt | dd/mm/yyyy |
| 4 | DATN-ST-001 | System Test Cases / kịch bản E2E (web + API; có thể kèm file Excel/MD) | Tiếng Việt | dd/mm/yyyy |
| 5 | DATN-TS-001 | Test suite / thứ tự thực thi (nếu nhóm tổ chức theo suite) | Tiếng Việt | dd/mm/yyyy hoặc N/A |
| 6 | DATN-DF-001 | Defect log (export từ GitHub Issues / Google Sheet / tương đương) | Tiếng Việt | Cập nhật theo đợt test |
| 7 | DATN-TR-001 | Báo cáo kiểm thử (tóm tắt phạm vi, môi trường, kết quả Pass/Fail, lỗi mở) | Tiếng Việt | dd/mm/yyyy |

---

## Ghi chú

- **Unit Test Plan** riêng: nếu giảng viên không yêu cầu tách, có thể gộp mô tả unit test vào **Test Plan** (mục 3) và báo cáo cuối — ghi rõ trong cột *Deliverables* hoặc đánh dấu N/A cho dòng tương ứng.
- **Ngày giao:** dùng định dạng **dd/mm/yyyy** (đồ án Việt Nam). Biểu mẫu Excel FSOFT có ghi *YY-MM-DD* khi khách hàng Nhật — **không bắt buộc** cho nhóm DATN.
- Các dòng có nội dung trong `<>` trên mẫu gốc là **hướng dẫn điền** — bản này đã thay bằng nội dung cụ thể; khi nộp bài chỉ cần điền **ngày** và bổ sung link file nếu thầy yêu cầu.
