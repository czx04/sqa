# 8. TEST MILESTONES

Các **mốc kiểm thử** dưới đây dùng để báo cáo tiến độ nhóm **DATN** (4 người). **Effort (pd)** = person-day (người-ngày), có thể điều chỉnh theo lịch thực tế. **Ngày** điền khi nhóm chốt kế hoạch.

---

## Bảng mốc

| Milestone Task | Effort (pd) | Start Date | End Date |
|----------------|-------------|------------|----------|
| Hoàn thiện Unit Test Plan / mô tả unit test trong Test Plan (nếu không tách file riêng) | 2 | dd/mm/yyyy | dd/mm/yyyy |
| Review và cập nhật phần unit test trong Test Plan | 1 | dd/mm/yyyy | dd/mm/yyyy |
| Tạo / bổ sung Unit Test case và script (`*.spec.ts` trọng yếu) | 5 | dd/mm/yyyy | dd/mm/yyyy |
| Review và cập nhật Unit TC / script | 2 | dd/mm/yyyy | dd/mm/yyyy |
| Hoàn thiện Test Plan (bộ `.md` / nội dung theo mẫu FSOFT) | 3 | dd/mm/yyyy | dd/mm/yyyy |
| Review và cập nhật Test Plan | 1 | dd/mm/yyyy | dd/mm/yyyy |
| Tạo Integration Test case / script (API + DB + sandbox) | 6 | dd/mm/yyyy | dd/mm/yyyy |
| Review và cập nhật Integration TC / script | 2 | dd/mm/yyyy | dd/mm/yyyy |
| Tạo System Test case / kịch bản E2E (web + luồng nghiệp vụ) | 8 | dd/mm/yyyy | dd/mm/yyyy |
| Review và cập nhật System TC / script | 2 | dd/mm/yyyy | dd/mm/yyyy |
| Thực thi Unit test / chạy script Jest | 3 | dd/mm/yyyy | dd/mm/yyyy |
| Thực thi Integration test | 5 | dd/mm/yyyy | dd/mm/yyyy |
| Thực thi System test | 8 | dd/mm/yyyy | dd/mm/yyyy |
| Tổng hợp defect log và **báo cáo kiểm thử** | 3 | dd/mm/yyyy | dd/mm/yyyy |

---

## Ghi chú

- **Thứ tự** có thể **chồng lấn** (ví dụ vừa viết System TC vừa chạy Integration test theo module) — điều chỉnh cột Start/End cho khớp burndown nhóm.
- Tổng effort trong bảng là **ước lượng tham khảo** (~50 pd); chia cho 4 người và số tuần thực tế để kiểm tra khả thi.
- Nếu giảng viên không yêu cầu **Unit Test Plan** riêng: gộp hai dòng đầu thành một mốc *“Chốt phạm vi unit test trong Test Plan”* và ghi **N/A** cho effort dòng thừa.
